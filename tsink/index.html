Built for performance.
Designed for simplicity.

Everything you need for time-series data, nothing you don't.

Gorilla Compression

Facebook's Gorilla algorithm achieves ~1.37 bytes per data point using delta-of-delta timestamps and XOR-based value encoding.

Thread-Safe Concurrency

Lock-free reads with concurrent writes via configurable worker pools. Built on DashMap, parking_lot, and crossbeam.

Dual Storage Modes

Run fully in-memory for maximum speed or persist to disk with memory-mapped files for zero-copy reads and automatic recovery.

Write-Ahead Logging

WAL ensures crash recovery with configurable sync modes and buffer sizes. Choose between periodic sync for throughput or per-append sync for strongest durability.

Multi-Dimensional Labels

Tag data points with key-value labels for flexible querying. Filter by any combination of labels with efficient binary marshaling.

Container-Aware

Automatic cgroup detection for CPU and memory limits. Optimal resource usage in Docker, Kubernetes, and other containerized environments.

Rich Querying

Downsampling, aggregation (Sum, Avg, Min, Max, Last), pagination, and time-range filtering built in. No external query engine needed.

Time-Based Partitioning

Automatic partition management with configurable durations. Old partitions expire based on retention policy. Out-of-order writes handled gracefully.

basic_usage.rs 
use tsink::{DataPoint, Row, Storage, StorageBuilder, TimestampPrecision};

fn main() -> Result<(), Box<dyn std::error::Error>> {
    // Create an in-memory storage instance
    let storage = StorageBuilder::new()
        .with_timestamp_precision(TimestampPrecision::Seconds)
        .build()?;

    // Insert data points
    let rows = vec![
        Row::new("cpu_usage", DataPoint::new(1600000000, 45.5)),
        Row::new("cpu_usage", DataPoint::new(1600000060, 47.2)),
        Row::new("cpu_usage", DataPoint::new(1600000120, 42.1)),
    ];
    storage.insert_rows(&rows)?;

    // Query the data
    let points = storage.select(
        "cpu_usage", &[], 1600000000, 1600000121
    )?;

    for p in &points {
        println!("ts={}, val={}", p.timestamp, p.value);
    }

    storage.close()?;
    Ok(())
}
persistent_storage.rs 
use tsink::{StorageBuilder, Storage, WalSyncMode};
use std::time::Duration;

fn main() -> Result<(), Box<dyn std::error::Error>> {
    // Persistent storage with WAL and retention policy
    let storage = StorageBuilder::new()
        .with_data_path("./tsink-data")
        .with_partition_duration(Duration::from_secs(3600))
        .with_retention(Duration::from_secs(7 * 24 * 3600))
        .with_wal_buffer_size(8192)
        .with_wal_sync_mode(WalSyncMode::Periodic(Duration::from_secs(1)))
        .with_max_writers(4)
        .with_write_timeout(Duration::from_secs(30))
        .build()?;

    // Data survives application restarts.
    // WAL ensures durability on crash.
    // Retention auto-cleans partitions older than 7 days.

    storage.close()?;
    Ok(())
}

aggregation.rs 
use tsink::{Aggregation, QueryOptions, Storage, StorageBuilder};

fn main() -> Result<(), Box<dyn std::error::Error>> {
    let storage = StorageBuilder::new().build()?;

    // ... insert data ...

    // Downsample + aggregate: 2-second buckets with average
    let opts = QueryOptions::new(1_000, 5_000)
        .with_downsample(2_000, Aggregation::Avg)
        .with_pagination(0, Some(10));

    let buckets = storage.select_with_options("cpu", opts)?;

    // Available aggregations: Sum, Min, Max, Avg, Last
    for dp in &buckets {
        println!("bucket ts={}, avg={}", dp.timestamp, dp.value);
    }

    storage.close()?;
    Ok(())
}

Fully configurable.

Tune every aspect of the storage engine via the builder API.

Option Description Default
with_data_path(path) Directory for persistent storage None (in-memory)
with_retention(dur) How long to keep data before auto-cleanup 14 days
with_timestamp_precision(p) Nanoseconds, Microseconds, Milliseconds, or Seconds Nanoseconds
with_max_writers(n) Maximum concurrent write workers CPU count
with_write_timeout(dur) Timeout for write operations 30 seconds
with_partition_duration(dur) Time range covered by each partition 1 hour
with_wal_enabled(bool) Enable write-ahead logging for durability true
with_wal_buffer_size(n) WAL buffer size in bytes 4096
with_wal_sync_mode(mode) WAL fsync strategy: Periodic(Duration) or PerAppend Periodic(1s)
